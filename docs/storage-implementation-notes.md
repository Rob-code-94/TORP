# Firebase Storage Implementation Notes

This document captures the implemented storage baseline and rollout controls without modifying the planning artifact.

## Implemented foundation

- Shared storage entities are defined in `types.ts` (`AssetRef`, `AssetVersion`, `StorageDeliveryLink`, lifecycle status).
- Tenant-safe storage key taxonomy is centralized in `lib/storagePaths.ts`.
- Project asset storage pathing now resolves under `tenants/{tenantId}/...`.
- `storage.rules` now enforces tenant isolation on the `tenants/{tenantId}` prefix.
- Firebase Storage client runtime is wired in `lib/firebase.ts` and initialized through `lib/storageRuntime.ts`.
- Project storage adapter is connected to Firebase-aware pathing (`lib/firebaseProjectAssetStorageAdapter.ts`).
- Upload status state machine guardrails are implemented in `lib/storageUploadState.ts`.
- Delivery security primitives are implemented in Cloud Functions:
  - `createStorageDeliveryLink`
  - `revokeStorageDeliveryLink`

## Delivery model

- Internal access: authenticated app runtime through Firebase Storage SDK and tenant-prefixed paths.
- External access: short-lived signed URL generated by callable function with tenant scope and role checks.
- Revocation model: link revocation writes to `storageDeliveryLinks` in Firestore and blocks future issuance lifecycle.

## Cross-module data hooks

- Planner items can track attachment asset references (`attachmentAssetIds`).
- Crew profiles can track media asset references (`mediaAssetIds`).
- Invoices and proposals can track attachment asset references (`attachmentAssetIds`).
- Deliverables can track approved version and issued delivery links.

## Upload state model

- Canonical states: `queued`, `uploading`, `processing`, `ready`, `failed`, `archived`.
- Allowed transitions are enforced by `canTransitionAssetStatus` and `assertAssetStatusTransition`.

## Mobile and layout guardrails

- New storage fields are model-level and do not introduce page-level horizontal overflow.
- Existing table surfaces continue using local `overflow-x-auto` containers instead of page-level scrolling.

## MVP rollout status
